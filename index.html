Quadnative Engine Architechture Docs

This class provide instance of QuadnativeEngine by static method get_instance (quad_config). This class contains An Interpreter, a Lexer and a parser.

  • Constructor

    QuadnativeEngine(quad_config)

    This is unusual constructor for user. But it used inside QuadnativeEngine class to creat get instance (quad_config).

    quad_config : This argument refers to a QuadnativeConfig class.

  • Functions
    • get_instance(quad_config)
      This is a startatic method and it used as QuadnativeEngine.get_instance (config) Its return an instance of QuadnativeEngine. This method follow singleton design pattern.
      quad_config : This argument refers to a QuadnativeConfig class.
    • render_quad (filename="index.q")
      This method is used to parse an 'q' file and return rendered html text.
      filename : This argument refers to a filename (type of string).
    • get_config()
      This method return QuadnativeConfig instance from QuadnativeEngine.get_config().

This class provide configuration of QuadnativeEngine. Such as configuration of Language Translation, configuration files and library of dependent 'q' file.

  • Constructor

    QuadnativeConfig(config,setting)

    This constructor create a new QuadnativeConfig class instance by taking filename or dictionary as config argument.

    config : This argument refers to a dictionary data or a json file configuration.
    If config is dictionary then key and value must be. 

                      lang:[('Tag','filename')]
                  lib:['path1','path2',.....]
                  config:'filename'
    If config is json file then key and value must be. 

                      {
                    name:"QuadnativeEngine",
                    ver:"1.0",
                    lang:[('Tag','filename')],
                    config :"filename",
                    lib:['path1','path2',.....]
                  }
    setting : This argument refers to instance of QuadnativeSetting.

    sample: 
                      setting=QuadnativeSetting()
                  config={
                    'lang':[
                      ('En','lang.En.q'),
                      ('Es','lang.Es.q')
                    ],
                    lib:['path1','path2',.....],
                    config:'configure.q'
                  }
                  q_config=QuadnativeConfig(config,setting)

  • Functions
    • get_setting()
      This method return QuadnativeSetting instance from QuadnativeConfig class.
    • get_langs()
      This method return list of languages (Tag,File)
    • get_config()
      This method return configuration filename.
    • get_libs()
      This method return libraries folder.
    • add_lib(path='path')
      This method add library folder in lib property.
      path : This argument refers to a folder path of 'q' files.
    • add_lang(lang=('En', 'lang.En.q'))
      This method add language suport in langs property.
      lang : This argument refers to a tuple with a tag and file name.
    • set_config(name='configure.q')
      This method change property of config.
      name : This argument refers to a configurable 'q' file.

This class provide control over QuadnativeEngine configuration such as print log adding external variables such as Server.Id , Cookie.Time etc. We can also add external function and we can change python command module.

  • Constructor

    QuadnativeSetting()

    This constructor create a new QuadnativeSetting class instance.

  • Functions
    • add_global_var(varname,value)
      this method add global variable in environment of QuadnativeEngine.Its take varname as variable name and value take type of Value, ArrayList and Dictionary.
      varname : This argument refers to a variable name.
      value: This argument refers to value of type Value, ArrayList and Dictionary.

      sample: 
                         setting= QuadnativeSetting()
                   setting.add_global_var("int_var", Value(9))
                   setting.add_global_var("string_var",Value("Hello"))
                   setting.add_global_var("float_var",Value(4.6))
                   setting.add_global_var("dictionary_var",Dictionary({'key':'value'}))
                   setting.add_global_var("array_var",ArrayList([9,1,2]))
    • add_function (functionname, function)
      This method add function in environment of QuadnativeEngine. Its take functionname as name of defined function and function as a type of Function.
      functionname : This argument refers to a function name.
      function : This argument refers to a Function.

      sample: 
                         def something():
                      print("hello world")
                   setting= QuadnativeSetting()
                   setting.add_function("Call_me",something)  
      //now something function called by Call_me
