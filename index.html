Quadnative Engine Architechture Docs

This class provide instance of QuadnativeEngine by static method get instance (quad_config). This class contains An Interpreter, a Lexer and a parser.

  • Constructor

    QuadnativeEngine(quad_config)

    This is unusual constructer for user. But it used inside QuadnativeEngine class to creat get instance (quad_config).

    quad_config : This argument refers to a QuadnativeConfig class.

  • Functions
    • get_instance(quad_config)
      This is a startatic method and it used as QuadnativeEngine.get_instance (config) Its return an instance of QuadnativeEngine. This method follow singleton design pattern.
      quad_config : This argument refers to a QuadnativeConfig class.
    • render_quad (filename="index.q")
      This method is used to parse an 'q' file and return rendered html text.
      filename : This argument refers to a filename (type of string).
    • get_config()
      This method return QuadnativeConfig instance from QuadnativeEngine.get_config().

This class provide configuration of QuadnativeEngine. Such as configuration of Language Translation, configuration files and library of dependent 'q' file.

  • Constructor

    QuadnativeConfig(config)

    This constructor create a new QuadnativeConfig class instance by taking filename or dictionary as config argument.

    config : This argument refers to a dictionary data or a json file configuration.
    If config is dictionary then key and value must be. 
                      lang:[('Tag','filename')]
                  lib:['path1','path2',.....]
                  config:'filename'
    If config is json file then key and value must be. 
                      {
                    name:"QuadnativeEngine",
                    ver:"1.0",
                    lang:[('Tag','filename')],
                    config :"filename",
                    lib:['path1','path2',.....]
                  }
